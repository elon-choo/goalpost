Claude Code 플러그인 · v0.3.0

당신의 AI 코딩 에이전트를 위한 규율 있는 프로젝트 매니저. A disciplined project manager for your AI coding agent.

프로젝트를 작고 검증 가능한 목표로 쪼개고, 무인으로 빌드하며, 강력하지만 위험한 모델을 짧은 고삐에 묶어둡니다. It breaks a project into small, verified goals, builds them hands-off, and keeps a powerful but reckless model on a short leash.

긴 작업은 두 가지로 실패합니다. Goalpost는 둘 다 막습니다.Long builds fail two ways. Goalpost fixes both.

에이전트는 완료되지 않은 걸 완료라 표시하고 재시작 후 맥락을 잊는 식으로 폭주하거나, 매 단계마다 당신에게 물어보며 멈춰섭니다. 하나의 장부(ledger)와 두 가지 규칙이 두 구멍을 모두 막습니다. An agent either runs away, marking things done that aren't and forgetting the thread after a restart, or it stalls, asking you at every step. One artifact (a ledger) and two rules close both gaps.

가짜 "완료" 없음No fake "done"

모든 목표는 실제 증거(실행된 테스트, 통과한 빌드)로 검증된 뒤에만 다음으로 넘어갑니다. "괜찮아 보이니 넘어가자"는 없습니다.Every goal is checked against first-party evidence, a test that ran or a build that passed, before it advances. Never "looks good, moving on."

알맞은 비용Right-sized cost

각 목표는 감당 가능한 가장 저렴한 모델로 실행됩니다. 비싼 최상위 모델은 어렵고 중요한 일에만 써서, 반복 작업에 최고가를 내지 않습니다.Each goal runs on the cheapest capable model. The expensive flagship is reserved for hard, critical work, so boilerplate never runs at top rates.

기본이 안전Safe by default

파괴적 작업은 샌드박스에 격리되고 게이트를 거칩니다. 0.3.0에서는 안전 계층 자체를 실제 사고에 대입해 측정하고 강화했으며, 한계도 정직하게 밝힙니다.Destructive work is sandboxed and gated. In 0.3.0 the safety layer was itself measured against real incidents, then hardened, and it's honest about its limits.

작동 방식How it works

목표로 쪼개고, 무인으로 하나씩 처리합니다.Plan into goals, then drain them hands-off.

하나의 LEDGER.md가 진실의 원천이라, 작업은 컨텍스트가 압축돼도 이어집니다. "계속해"라는 말만 들은 새 세션이 정확히 멈춘 지점에서 다시 시작합니다.A single LEDGER.md is the source of truth, so the work is compaction-proof. A fresh session told "keep going" picks up exactly where the last one stopped.

모드 1 · 계획Mode 1 · Plan

프로젝트를 분해Decompose the project

작업을 스테이지로, 다시 원자적 목표로 나눕니다. 각 목표는 어울리는 모델 티어와 완료 정의(DoD)를 갖습니다. 관찰 가능한 통과/실패 명령이거나, 새 리뷰어가 채점하는 체크리스트입니다.It splits the work into stages, then atomic goals. Each carries the model tier it deserves and a Definition of Done: a command with an observable pass or fail, or a checklist a fresh reviewer scores.

"이 프로젝트를 스테이지 × 목표 로드맵으로 쪼개줘.""Break this project into a stage × goal roadmap."
모드 2 · 실행Mode 2 · Run

빌드하고, 검증하고, 전진Build, verify, advance

목표마다: 디스패치한 뒤 완료 정의를 직접 다시 실행합니다. 실패하면 티어를 한 단계 올려 증거와 함께 재시도하고(최대 3번), 통과한 검사에서만 닫습니다. 모든 스테이지 경계는 독립 리뷰가 게이트합니다.Per goal: dispatch, then independently re-run its Definition of Done. On failure it escalates a tier and retries with the evidence, up to three strikes, and closes only on a passing check. An independent review gates every stage boundary.

"내가 없는 동안 원장을 무인으로 실행해줘.""Run the ledger hands-off while I'm away."
모델 라우팅Model routing

태양, 지구, 달. 목표는 자기에게 맞는 모델을 받습니다.Sun, Earth, Moon. A goal gets the model it deserves.

Codex의 세 티어는 천체 이름을 땄고, Goalpost는 각 목표를 위험 반경 × 변동성으로 배정합니다. 보안이나 데이터 손실 목표는 최상위 모델에 고정되어 비용 절약을 이유로 절대 강등되지 않으며, 반복 작업은 그 모델을 낭비하지 않습니다.Codex's three tiers are named for celestial bodies, and Goalpost routes each goal by its blast radius × variance. A security or data-loss goal is pinned to the flagship and can never be downgraded to save money; boilerplate never wastes it.

solgpt-5.6-sol · 천장(최상위)the ceiling

어렵고 중요한 것만: 보안, 인증, 결제, 파괴적 마이그레이션, 광범위 리팩터, 최종 리뷰. 예약석이며, 일상 작업엔 쓰지 않습니다.Hard and critical only: security, auth, payments, destructive migrations, cross-cutting refactors, final review. Reserved, never routine.

terragpt-5.6-terra · 주력the workhorse

기본값. 기능 구현, API·DB 로직, 일반 버그 수정, 중간 규모 리뷰.The default. Feature implementation, API and DB logic, ordinary bug fixes, medium reviews.

lunagpt-5.6-luna · 가벼운 작업light work

보일러플레이트, 픽스처, 패턴 테스트, 문서, 기계적 편집. 답이 뻔한 곳에서 싸고 빠르게.Boilerplate, fixtures, tests on a pattern, docs, mechanical edits. Cheap and fast where the answer is obvious.

실패한 목표는 실패 증거를 달고 한 티어 위로 올라갑니다. 같은 걸 반복하지 않습니다.A failed goal escalates one tier up with its failure evidence attached. It doesn't just repeat the same one.

파괴 행위 안전장치 · 측정하고, 강화함Destructive-action safety · measured, then hardened

광범위한 지시에 파일을 지워버리는 최상위 모델을, 벽으로 막았습니다.A flagship that deletes things on a broad instruction, walled off.

GPT-5.6 Sol에는 실제 보고된 사고가 있습니다: 하룻밤 새 모든 유료 구독을 취소하고, 데이터베이스 전체를 삭제하고, 변수를 잘못 읽어 폴더를 지웠습니다. 0.3.0은 가드레일이 이 중 무엇을 실제로 잡는지 측정하고, 막을 수 있는 구멍을 닫았으며, 필터가 못 막는 것은 정직하게 남겨둡니다.GPT-5.6 Sol has real reported incidents: canceling every paid subscription overnight, deleting a whole database, wiping folders after misreading a variable. Version 0.3.0 measured which of these the guardrail actually caught, closed the gaps it could, and stays honest about the ones a filter can't.

+

필터가 이제 잡는 것What the filter now catches

  • 리터럴 파괴 명령: rm -rf, DROP TABLE, DELETE FROM, 강제 푸시Literal destructive commands: rm -rf, DROP TABLE, DELETE FROM, force-push
  • 빠른 레인의 최상위 모델: full access에서의 명시적 gpt-5.6-sol 호출The flagship on the fast lane: an explicit gpt-5.6-sol call at full access
  • 대량 변이 호출: cancelAllSubscriptions(), deleteMany(), WHERE 없는 UPDATE … SETMass-mutation calls: cancelAllSubscriptions(), deleteMany(), an UPDATE … SET with no WHERE
!

정직하게, 못 잡는 것What it honestly does not

  • 자연어 우회, 예: "유저 테이블의 모든 행을 지워"Natural-language paraphrase, such as "wipe all rows in the users table"
  • WHERE 1=1 같은 항상 참인 조건A self-true predicate like WHERE 1=1
  • 런타임의 무엇이든: 필터는 디스패치만 보고, 그 뒤 실행되는 셸 명령은 못 봅니다Anything at runtime: the filter only sees the dispatch, not the shell commands that follow
토큰 필터는 첫 번째 벽이 아니라 두 번째 벽입니다. 진짜 방어는 최소 권한입니다: 최상위 모델과 파괴 가능 목표는 제한된 샌드박스에서 실행되며 절대 full access를 받지 않고, 실제 데이터 파괴에는 사람 게이트가 붙습니다. 최소 권한이 벽이고, 필터는 그 위에 얹힙니다. The token filter is the second wall, not the first. The durable control is least privilege: the flagship and any destructive-capable goal run in a restricted sandbox, never at full access, plus a human gate on real data destruction. Least privilege is the wall; the filter rides on top of it.

Claude Code 플러그인이자 자체 마켓플레이스입니다. 엔지니어링 목표에는 한 가지가 필요합니다: MCP 서버로 연결된 당신의 Codex 로그인(codex login). Goalpost는 자격증명을 절대 포함하거나 요구하지 않습니다.It's a Claude Code plugin and its own marketplace. The engineering goals want one thing: your own Codex login (codex login) exposed as an MCP server. Goalpost never bundles or asks for credentials.

이 페이지의 모든 주장은 실제 배포된 동작에 근거합니다. 안전 계층은 심층 방어를 돕는 장치이지 보장이 아닙니다: 광범위하게 지시된 모델의 피해 반경을 줄이지만, 파괴를 불가능하게 만들지는 않습니다. 에이전트가 건드려도 괜찮은 작업에 쓰고, 최소 권한을 유지하며, 실제 데이터 파괴는 사람이 게이트하세요.Every claim on this page traces to shipped behavior. The safety layer is defense-in-depth, not a guarantee: it reduces the blast radius of a broadly-instructed model, it does not make destruction impossible. Run it on work you can afford to have an agent touch, keep least privilege on, and gate real data destruction to a human.
